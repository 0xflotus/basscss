Getting Started

Vanilla CSS

Include basscss.min.css out-of-the-box or use the unminified basscss.css and edit it how you see fit. For lighter-weight projects use basscss-lite.min.css, which only includes base type and layout styles. Manually adjust the font stack and theme styles to customize the look and feel.

Forms

Forms use default base styles for their structural appearance and thematic styles for colors and borders. This creates a rhythmic consitency among all forms while allowing thematic differences where needed. Form elements do not rely on nested styles or markup structure, so utility styles can be used for contextual adjustments.

Inline Forms

By default, form elements are display inline. Use the default .input theme or create custom themes to control the look. 

<form>
  <label class="">Search</label>
  <input type="text" class="input">
  <button class="button-blue">Go</button>
</form>

Stacked Forms

Use .block and other utilities to stack form elements. This allows for a lot of flexibility with form layouts. 

<form class="sm-col-6">
  <label>Email Address</label>
  <input type="text" class="block full-width input">
  <label>Password</label>
  <input type="password" class="block full-width input">
  <label>Select</label>
  <select class="block full-width input">
    <option>Option 1</option>
    <option>Option 2</option>
    <option>Option 3</option>
    <option>Option 4</option>
    <option>Option 5</option>
  </select>
  <label class="block full-width mb2">
    <input type="checkbox" checked>
    Remember me
  </label>
  <button class="button-blue">Sign In</button>
</form>

Utilities

Utilities are single-purpose styles to handle common layout tasks.

Layout

Use the following layout utilities to alter default HTML layout.

.inline       { display: inline }
.block        { display: block }
.inline-block { display: inline-block }

.overflow-hidden { overflow: hidden }

.clearfix:before,
.clearfix:after {
  content: &quot; &quot;;
  display: table
}
.clearfix:after { clear: both }

.fit { max-width: 100% }

.half-width { width: 50% }
.full-width { width: 100% }

Floats

These utilities are available in mobile-first breakpoint-based variations.

Prefix Description
(no prefix) Not scoped to a breakpoint
sm- --breakpoint-sm (default: min-width 40em)
md- --breakpoint-md (default: min-width 52em)
lg- --breakpoint-lg (default: min-width 64em) 
.left  { float: left }
.right { float: right }
@media (--breakpoint-sm) {
  .sm-left  { float: left  }
  .sm-right { float: right }
}
@media (--breakpoint-md) {
  .md-left  { float: left  }
  .md-right { float: right }
}
@media (--breakpoint-lg) {
  .lg-left  { float: left  }
  .lg-right { float: right }
}

Margins

Margin and padding utilities are based on a global white space scale defined with variables. These utilities use a shorthand naming convention.

Shorthand Description
m Margin
p Padding
t Top
r Right
b Bottom
l Left
x X-axis (left and right)
y Y-axis (top and bottom)
n Negative (margin only)
1 --space-1 (default .5rem)
2 --space-2 (default 1rem)
3 --space-3 (default 2rem)
4 --space-4 (default 4rem) 
.m0  { margin:        0 }
.mt0 { margin-top:    0 }
.mr0 { margin-right:  0 }
.mb0 { margin-bottom: 0 }
.ml0 { margin-left:   0 }

.ml1 { margin-left:   var(--space-1) }

Responsive States

Responsive show/hide utitlies use the global mobile-first approach.

.sm-show, .md-show, .lg-show {
  display: none !important
}

@media (--breakpoint-sm) {
  .sm-show { display: block !important }
}

@media (--breakpoint-md) {
  .md-show { display: block !important }
}

@media (--breakpoint-lg) {
  .lg-show { display: block !important }
}

.hide { display: none !important }

@media (--breakpoint-sm) {
  .sm-hide { display: none !important }
}

@media (--breakpoint-md) {
  .md-hide { display: none !important }
}

@media (--breakpoint-lg) {
  .lg-hide { display: none !important }
}

Modules

Modules are more complex layout styles that are tied to markup structure.

Grid

The utility-based grid allows a lot of flexibility in creating complex layouts. Start by using a .clearfix container. Optionally use a .container to set a max-width. 

<div class="container">
  <div class="clearfix">
  </div>
</div>

Add columns using the .left and grid width .col-N classes.

.left.col-6
.left.col-6
<div class="clearfix">
  <div class="left col-6 outline-blue">.left.col-6</div>
  <div class="left col-6 outline-blue">.left.col-6</div>
</div>

Use breakpoint-prefixed float and column utilities to change the grid at different screen widths.

.sm-left.sm-col-6.md-col-5.lg-col-4
.sm-left.sm-col-6.md-col-7.lg-col-8
<div class="clearfix">
  <div class="sm-left sm-col-6 md-col-5 lg-col-4 outline-blue">.sm-left.sm-col-6.md-col-5.lg-col-4</div>
  <div class="sm-left sm-col-6 md-col-7 lg-col-8 outline-blue">.sm-left.sm-col-6.md-col-7.lg-col-8</div>
</div>

Use padding and negative margin utilities to create gutters based on the white space scale.

.px2
.px2
<div class="clearfix mxn2">
  <div class="sm-left sm-col-6 md-col-5 lg-col-4 px2 outline-blue"><div>.px2</div></div>
  <div class="sm-left sm-col-6 md-col-7 lg-col-8 px2 outline-blue"><div>.px2</div></div>
</div>
